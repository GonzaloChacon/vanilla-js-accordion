JS Accordion

Some notes on the development decisions:

  • The task specifies implementing ES6, reason for which I've implemented ES6 methods instead of more standardized one, for example '[].includes()' instead of the old way '[].indexOf()', or '[].forEach()' instead of 'for of loop'.
  • Since UI wasn't specified, and I was given a mock, I understood you expect me to deliver the same as the mock, I hope it's okay. I had to open the sketch file into a program to convert it, this is what I saw, not sure if some style was lost in the conversion, I hope not.
  • Grunt tasks are configured, however I'm sure they can be optimized, it has been a couple of years since the last time I've implemented Grunt and Gulp.
  • Inside 'index.js' file I've created a module to handle the component class and instantiation, and a module to contain animations. Usually when working in bigger project you separate those in different files but since this is quite small I consider it's better to keep them in the same file in order to be able to see them better together.
  • Most of the animations are made with CSS transitions but since the idea was to handle JS I've handle some from there implementing requestAnimationFrame API to better sync animations into the browser's render cycle.
    Usually I solve accordions just with CSS, implementing the label and a checkbox technique is enough.
  • If you run 'npm audit' you will see some warnings. I had those in other projects, they appear a couple of days ago, so far it seems there is not a version to get ride of those.
  • DISCLAIMER: I've developed this project in a MacBook and I was able to test the accordion in Safari, Firefox and Chrome, but I do not have access to a windows machine with Edge and IE11 to debbug right now. If required please let me know.

I hope you like the solution for the task. Please let me know what you think. :)
